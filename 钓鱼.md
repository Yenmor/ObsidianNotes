# 创建钓鱼网站，诱骗用户输入密码

## 常用手段

**1.  字符替换**

- **经典例子：**
    
    - `rn` 替换 `m`： 将 `www.amazon.com` 变成 `www.arnazon.com`（注意看，是“r n”不是“m”）。
        
    - `my` 替换 `ny`： 将 `www.paypal.com` 变成 `www.paypai.com`（看起来像“l”，但其实是“i”和“l”的组合？不，这里是用“ai”替换“ay”，但更常见的是用数字 `1` 替换字母 `l`）。
        
    - `0` (数字零) 替换 `O` (大写字母O)： `www.g00gle.com` 而不是 `www.google.com`。
        
    - `1` (数字一) 替换 `l` (小写字母L)： `www.micros1oft.com`。
        
    - `vv` 替换 `w`： `www.vvisa.com` 模仿 `www.visa.com`。
		
	- 使用西里尔字母的 `а` (U+0430) 来替换拉丁字母的 `a` (U+0061)。
**2. 子域名欺骗**  
利用人们对URL结构的误解。URL的阅读顺序是从右到左权威性递减。

- **例子：**
    
    - 真实网址： `login.github.com` (这是GitHub的合法子域名)
        
    - 钓鱼网址： `github.secure-login.com` 或 `github.com.youraddress.hackersite.com`
        
    - **解析：** 在第二个钓鱼网址中，真正的域名是 `hackersite.com`，前面的所有部分都是它的子域名。你只看到了开头的“[github.com](https://github.com/)”，却忽略了后面真正的归属。
        

**3. 短网址服务**  

利用Bitly、TinyURL等服务将长网址缩短，隐藏真实的恶意地址。

- **防范：** 在点击短链接前，可以使用一些在线扩展或服务（如“检查短链接”）来预览其真实目的地。

**4. 隐藏链接**  

在邮件或网页中，将一个文本（如“点击这里查看账单”）链接到一个完全不同的网址。


**5. 界面模仿（UI克隆）**  

攻击者1：1地复制真实网站的整个界面，包括Logo、布局、颜色、字体，甚至版权信息和脚注，看起来和真的一模一样。
