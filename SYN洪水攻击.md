# 1. 概述

	SYN洪水攻击显然是DOS或DDOS攻击的一种实现方式，在tcp第二次握手中故意不进行最终握手
	以消耗服务器的缓冲区，达到使服务器拒绝服务的目的
# 前置 ：[[TCP三次握手]]

攻击者利用了三次握手的第二个阶段。在收到服务器的`SYN-ACK`回应后，**攻击者不予理睬，不发送最终的`ACK`包进行确认**。

这样一来，服务器的“半开连接”会一直停留在“半连接队列”中。这个队列的大小是有限的。

攻击者会：

1. 伪造大量的IP地址，向目标服务器发送海量的`SYN`包。

2. 对于每一个`SYN`包，服务器都会善意地回应一个`SYN-ACK`，并在内存中创建一个半开连接，等待最终确认。

3. 攻击者永不发送确认`ACK`。


后果就是：

- 服务器的“半连接队列”被这些永不完成的半开连接迅速占满。
    
- 当队列满了之后，服务器将无法接受任何新的连接请求（包括正常的用户请求）。
    
- 正常用户发送的`SYN`包会被服务器丢弃或忽略，导致连接超时，无法访问服务。
